# Claude AI в репозитории MineGNK

## Что это

В репозитории настроен AI-ассистент **Claude**, который может:

- Автоматически ревьюить код в Pull Request'ах
- Выполнять задачи по запросу: менять текст, исправлять баги, добавлять функции
- Создавать Pull Request'ы из задач (issues)

## Как пользоваться

### 1. Создать задачу (issue) для Claude

Зайди в репозиторий > **Issues** > **New issue** и напиши задачу, упомянув `@claude`.

**Примеры:**

```
Заголовок: @claude Поменять цену H100

Тело:
Обнови цену сервера H100 с $12,300 на $15,000 в месяц.
Пересчитай hourly rate соответственно.
```

```
Заголовок: @claude Обновить FAQ

Тело:
Добавь новый вопрос в FAQ секцию:
Вопрос: "Do you offer annual contracts?"
Ответ: "Currently all plans are month-to-month. Contact us for custom arrangements."
Не забудь добавить переводы на русский и китайский.
```

```
Заголовок: @claude Исправить опечатку

Тело:
На главной странице в секции Features написано "monitorring" вместо "monitoring".
Найди и исправь во всех языках.
```

**Что произойдёт:**
1. Claude прочитает задачу
2. Создаст ветку и сделает изменения
3. Откроет Pull Request с описанием что и зачем поменял
4. Автоматически запустится код-ревью и CI (lint, tests, build)
5. Ты или кто-то из команды мержит PR после проверки

### 2. Попросить Claude в существующем issue или PR

Оставь комментарий с `@claude` в любом issue или PR:

```
@claude добавь перевод на китайский для этого текста
```

```
@claude объясни что делает этот код
```

```
@claude предложи как улучшить этот компонент
```

### 3. Автоматический код-ревью

Каждый PR автоматически проверяется Claude. Он оставит комментарии если найдёт:

- Баги или потенциальные ошибки
- Проблемы безопасности
- Несоответствие стилю кода
- Отсутствующие переводы

Ничего делать не нужно, ревью запускается сам.

## Что происходит после мержа PR

```
PR замержен в main
      |
      v
CI: lint -> test -> build
      |
      v
Deploy: сайт обновлён на сервере (~2-3 мин)
```

Деплой полностью автоматический. После мержа в `main` сайт обновится сам.

## Важно

- **Claude НЕ мержит PR сам** — всегда нужна ручная проверка и одобрение
- **Пиши задачи конкретно** — чем точнее описание, тем лучше результат
- **Один issue = одна задача** — не смешивай несколько изменений в одном issue
- **Проверяй PR перед мержем** — AI может ошибиться, всегда смотри diff
- **Английский для кода** — тело задачи можно писать на русском, но если указываешь конкретный текст для сайта, пиши его на том языке, на котором он должен появиться

## Быстрая шпаргалка

| Хочу | Что делать |
|------|-----------|
| Поменять текст на сайте | Issue: `@claude Поменяй текст X на Y в секции Z` |
| Исправить баг | Issue: `@claude Исправь [описание бага]` |
| Добавить функцию | Issue: `@claude Добавь [описание]` |
| Спросить про код | Комментарий в PR: `@claude что делает этот файл?` |
| Попросить ревью | Комментарий в PR: `@claude проверь этот PR` |
